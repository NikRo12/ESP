# ESP32-C3 Smart IoT Controller

Прошивка для **ESP32-C3 DevKitM-1**, превращающая плату в автономный IoT-контроллер с веб-интерфейсом. Устройство поднимает свою собственную WiFi-точку доступа, отдаёт HTML-страницу из встроенной файловой системы и предоставляет REST API и WebSocket для управления светодиодами, RGB-лентой и считывания значения с потенциометра в реальном времени.

Проект собран на **ESP-IDF** через **PlatformIO** и совместим с симулятором **Wokwi**, поэтому работает как на железе, так и в браузере без подключённой платы.

## Возможности

- **WiFi Access Point** (WPA2-PSK) — клиенту достаточно подключиться к сети, никакой роутер не нужен
- **Встроенный HTTP-сервер** на порту 80 с поддержкой CORS
- **WebSocket** на `/ws` для push-уведомлений о смене состояния — UI обновляется без поллинга
- **LittleFS** в отдельном разделе flash — UI (`index.html`) загружается прямо через браузер
- Управление двумя реле/светодиодами через физические кнопки **и** через API
- **WS2812 RGB-лента** (64 светодиода) с программным троттлингом обновлений (не чаще 1 раза в 50 мс)
- Чтение потенциометра через ADC с расчётом значения в процентах
- Сериализация полного состояния устройства в JSON

## Аппаратная часть

| Компонент | GPIO | Назначение |
|---|---|---|
| LED 1 | GPIO 3 | Управляется кнопкой 1 и `POST /switch1` |
| LED 2 | GPIO 4 | Управляется кнопкой 2 и `POST /switch2` |
| Кнопка 1 | GPIO 6 | Тоггл LED 1 |
| Кнопка 2 | GPIO 7 | Тоггл LED 2 |
| RGB (WS2812) | GPIO 5 | Лента из 64 светодиодов |
| Потенциометр | ADC1 CH0 (GPIO 0) | Аналоговый вход 0–100 % |

Распиновка для Wokwi описана в `diagram.json` — проект открывается «как есть» через расширение Wokwi для VS Code.

## Структура репозитория

```
ESP/
├── src/
│   ├── main.cpp           # точка входа app_main()
│   ├── Application.*      # склейка всех компонентов + основной tick-loop
│   ├── Server.*           # обёртка над esp_http_server (HTTP + WebSocket)
│   ├── endpoints.*        # обработчики REST-эндпоинтов
│   ├── Wifi.*             # инициализация WiFi в режиме AP
│   ├── Filesystem.*       # монтирование LittleFS
│   ├── Led.hpp            # GPIO-выход
│   ├── Button.*           # GPIO-вход с антидребезгом (polling)
│   ├── RGB.*              # драйвер WS2812 через led_strip
│   ├── AdcManager.*       # oneshot-режим ADC
│   ├── Potentiometer.*    # чтение значения и пересчёт в %
│   └── platform.*         # инициализация NVS / netif / event loop
├── managed_components/    # зависимости IDF Component Manager
│   ├── espressif__led_strip
│   └── joltwallet__littlefs
├── partitions.csv         # кастомная таблица разделов (1 МБ под LittleFS)
├── platformio.ini         # конфигурация сборки
├── diagram.json           # схема для симуляции в Wokwi
└── wokwi.toml             # привязка Wokwi к артефактам сборки
```

## HTTP API

Все ответы — `application/json` с заголовками CORS.

| Метод | Путь | Описание |
|---|---|---|
| `GET` | `/` | Отдаёт `index.html` из LittleFS. Если файл не загружен — заглушка со ссылкой на `/upload` |
| `GET` | `/upload` | HTML-форма для загрузки UI |
| `POST` | `/upload` | Записывает тело запроса в `/littlefs/index.html` |
| `POST` | `/switch1` | Переключает LED 1, возвращает новое состояние (`true`/`false`) |
| `POST` | `/switch2` | Переключает LED 2 |
| `GET` | `/state` | Полное состояние устройства в JSON |
| `POST` | `/rgb` | Принимает `{"r":0-255,"g":0-255,"b":0-255}`, применяет цвет с троттлингом |
| `WS` | `/ws` | WebSocket, на который сервер пушит JSON-состояние при каждом изменении |

Формат состояния:

```json
{
  "switch1": false,
  "switch2": true,
  "lightSensor": 47,
  "rgb": { "r": 255, "g": 128, "b": 0 }
}
```

## WiFi

После прошивки устройство поднимает точку доступа:

- **SSID:** `qwe`
- **Пароль:** `12345678`
- **Режим:** WPA2-PSK, до 4 клиентов
- **IP шлюза по умолчанию:** `192.168.4.1`

После подключения откройте в браузере `http://192.168.4.1/`.

> Учётные данные WiFi захардкожены в `main.cpp`. Для боевого использования вынесите их в `sdkconfig` или NVS.

## Сборка и прошивка

Нужен **PlatformIO** (CLI или расширение для VS Code).

```bash
# Сборка
pio run

# Прошивка через USB
pio run -t upload

# Монитор последовательного порта (115200)
pio device monitor
```

Окружение: `esp32-c3-devkitm-1`, framework `espidf`, скорость монитора и заливки — 115200.

## Симуляция в Wokwi

1. Установите расширение **Wokwi for VS Code**
2. Соберите проект: `pio run`
3. Откройте `diagram.json` — в нём уже есть плата, потенциометр и LED-матрица в качестве визуализации RGB-ленты
4. Запустите симуляцию (`F1` → `Wokwi: Start Simulator`)

Привязка артефактов сборки к симулятору описана в `wokwi.toml`.

## Загрузка веб-интерфейса

После первой прошивки LittleFS пустой, и `/` отдаёт заглушку. Чтобы залить UI:

1. Подключитесь к WiFi `qwe`
2. Откройте `http://192.168.4.1/upload`
3. Выберите HTML-файл (он должен быть самодостаточным — JS/CSS встроены)
4. Нажмите **Upload** — файл сохранится как `/littlefs/index.html` и будет отдаваться по корневому пути

Файл переживает перезагрузки и перепрошивки приложения (раздел `storage` не очищается при обновлении прошивки).
